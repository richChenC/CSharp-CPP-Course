# 第14课_异步并发与任务调试

## 本节目标
学会最基础的 `async/await` 思想，并知道异步程序应该怎么调试。

## 本节知识点
- `Task`
- `async`
- `await`
- `Task.Delay`
- 异步流程观察

## 重点理解
- 异步不是多线程的同义词
- `await` 的重点是“等任务完成后继续”
- 调试异步代码时，要看执行顺序，不要只看最终结果

## 调试训练重点
- 给异步方法入口和 `await` 后打断点
- 观察输出顺序
- 解释为什么某些输出不是线性同步顺序

## 本节小结
异步最容易把初学者绕晕，但只要盯住“调用前、等待中、恢复后”三个阶段，就能慢慢看懂。
