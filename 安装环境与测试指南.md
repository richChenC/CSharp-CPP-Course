# 安装环境并运行测试

当前 Codex 沙箱的联网提升权限审批服务异常，我无法直接替你下载安装。所以我已经把一键脚本放在本课程包根目录：

- 安装环境并测试课程.ps1

## 使用方法

在 Windows PowerShell 中进入课程目录，然后运行：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\安装环境并测试课程.ps1
```

脚本会做这些事：

1. 在当前工作区创建 .tools 目录。
2. 下载并安装本地 .NET SDK。
3. 下载并解压便携 C++ 编译器 w64devkit。
4. 编译测试全部 C# 示例。
5. 编译测试全部 C++ 示例。
6. 生成 真实编译测试报告.md。

## 不想重复安装

如果 .tools 里已经装好环境，可以运行：

```powershell
.\安装环境并测试课程.ps1 -SkipInstall
```

脚本默认不安装到系统目录，不修改系统 PATH，只在当前工作区使用本地工具。

## 2026-05-09 修复记录

脚本已修复 C++ 编译器下载逻辑：现在会通过 GitHub API 自动查找最新 release 里的 `w64devkit-x64-*.7z.exe`，不再使用固定 zip 地址。

如果 .NET SDK 已经安装成功，只需要重新运行：

```powershell
.\安装环境并测试课程.ps1
```

脚本会跳过已存在的 .NET SDK，继续下载 C++ 编译器并运行完整测试。

## 2026-05-09 进一步修复

C++ 编译器包是一个自解压 exe，直接执行时在部分环境里会出现“文件所在卷已被外部更改”的问题。脚本已改为先下载 7zr.exe，再用它解压 w64devkit，这样更稳。

如果你刚才已经下载了 w64devkit 文件，但解压失败，只需要重新运行：

```powershell
.\安装环境并测试课程.ps1
```

脚本会继续解压并完成后续测试。

## 这次修复的重点

C++ 下载逻辑已改成通过 GitHub API 自动查找最新的 x64 资产，并在下载后再次检查文件是否真的落盘，再执行解压。这样如果 .tools 里有旧残留，也不会误判。
