# 第15课_调试实战_崩溃排查与断点分析

## 本节目标
系统训练你排查 bug 的过程，而不是只看正确代码。

## 本节知识点
- 断点
- 单步执行
- 观察变量
- 崩溃前定位
- 从现象反推原因

## 重点理解
- 调试不是乱点按钮，而是有顺序地缩小问题范围
- 先确认输入，再确认中间状态，最后确认出错点
- 看到崩溃不要慌，先找“最后一个正常状态”

## 本节调试流程
1. 先运行程序，看报错现象。
2. 在关键变量赋值处打断点。
3. 逐步执行，确认哪一步开始偏离预期。
4. 找到根因后，再思考为什么肉眼一开始没看出来。

## 面试高频点
- 线上 bug 一般怎么定位
- 断点、日志、单步执行各适合什么场景
- 越界、空指针、未初始化变量怎么排查

## 本节小结
真正让人成长快的，不是一次写对，而是能快速查出为什么错。
