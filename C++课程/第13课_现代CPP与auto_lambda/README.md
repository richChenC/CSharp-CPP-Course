# 第13课_现代CPP与auto_lambda

## 本节目标
学会使用 `auto`、范围 `for`、Lambda 表达式，并理解现代 C++ 为什么更强调“写得更安全、更少重复”。

## 本节知识点
- `auto` 自动类型推导
- 范围 `for`
- Lambda 表达式
- `std::sort`
- `const` 引用的使用习惯

## 重点理解
- `auto` 不是“偷懒”，而是减少类型重复书写和降低出错概率
- Lambda 本质上是“可以临时定义的小函数”
- 调试时要重点看变量真实类型、容器元素变化、排序前后结果

## 老师式拆解
这一课不是为了让你背语法糖，而是让你理解现代 C++ 的代码为什么更短、更清晰。

学习顺序建议：
1. 先看示例程序输出了什么。
2. 再看 `auto` 推导出来的实际类型是什么。
3. 最后重点理解 Lambda 为什么能直接写在排序函数里。

## 调试训练重点
- 在 VS Code 里给 `numbers` 初始化后打断点
- 单步执行 `sort`
- 观察排序前后容器内容变化
- 看清 Lambda 的参数 `left` 和 `right` 在比较时分别是什么

## 面试高频点
- `auto` 的优点和风险是什么
- Lambda 捕获列表是什么
- `const auto&` 为什么常用在遍历里
- `std::sort` 的时间复杂度大概是多少

## 学习方式
1. 读讲义
2. 运行示例
3. 自己改排序规则
4. 故意写错一版再调试
5. 复述今天每个新语法的用途

## 练习目标
学完后你至少要能：
- 自己写出一个带 Lambda 的排序
- 说清 `auto` 和 `const auto&` 的区别
- 用调试器看懂排序前后数据变化

## 扩展思考
- 如果容器里装的是结构体，还能不能排序
- 如果想按多个字段排序，该怎么写 Lambda
- 如果排序结果不对，该先看比较函数还是先看原始数据

## 本节小结
现代 C++ 很多写法看起来“更高级”，但本质还是在做三件事：减少重复、提高可读性、降低 bug 概率。
