## 重点理解
- 菜单系统

## 老师式拆解
学习编程不要先死背概念，要先看懂“数据、流程、结构”这三件事。每节课都可以按这个顺序学：
1. 先看代码在做什么。
2. 再看每一行为什么这么写。
3. 最后想一想能不能换一种写法。

## 面试高频点
- 如何设计一个 C++ 小项目
- 为什么要拆成函数
- vector 和文件结合怎么做

## 学习方式
1. 读讲义。
2. 看示例。
3. 抄一遍。
4. 改一遍。
5. 关掉答案再写一遍。

## 练习目标
本节学完后，你至少要能：
- 说出核心概念的用途。
- 独立写出一个相似小例子。
- 看懂常见报错并定位问题。

## 扩展思考
- 这个知识点还能用在哪里？
- 如果数据更多，会不会更慢？
- 如果把它放到真实项目里，结构应该怎么变？

## 本节小结
这一课最重要的不是“记住一切”，而是形成正确的阅读代码习惯。把每行代码看明白，才算真正进入编程世界。
