# C# 与 C++ 课程学习指南

这份指南是整个课程的总说明，告诉你这套课怎么学、重点练什么、学完后应该达到什么状态。

## 一、这套课程包含什么

当前课程分成两条主线：

- `C#课程`：面向 C# 基础、进阶和调试能力训练
- `C++课程`：面向 C++ 基础、现代特性、资源管理和调试训练

你可以只学其中一条，也可以两条一起学。

## 二、两套课分别适合什么方向

### C#

更适合这些方向：

- Unity 开发
- Windows 桌面开发
- .NET 后台开发
- 企业级业务系统

### C++

更适合这些方向：

- 算法与竞赛
- 游戏引擎与图形
- 嵌入式与系统开发
- 高性能程序开发

## 三、这套课的教学风格

这套课不是“只给答案”的速成资料，而是按下面的方式组织：

1. 每节一个独立文件夹
2. 每节都配 `README.md`
3. 示例代码逐行写中文注释
4. 再配 `CodeWalkthrough.md` 做逐行拆解
5. 每节都带练习和课堂笔记模板
6. 关键课加入调试训练

## 四、你应该怎么学

建议你每一节课都按这个顺序：

1. 先读 `README.md`
2. 再运行示例程序
3. 对照代码注释理解逻辑
4. 再看 `CodeWalkthrough.md`
5. 完成 `Practice.md`
6. 把自己的理解、报错、调试过程写进 `Notes.md`

## 五、最重要的不是“看懂”，而是“能自己改”

很多人学编程最大的问题，不是没看过，而是：

- 只看不写
- 只抄不改
- 报错就绕过去

你要强迫自己做三件事：

1. 自己敲一遍
2. 故意改错一遍
3. 自己调试回来

## 六、你一定要练的调试能力

以后真正拉开差距的，不只是“会写代码”，而是：

- 会看报错
- 会下断点
- 会看变量变化
- 会一步步定位 bug

## 八、学完后能到什么程度

如果你认真学完并真正做完练习，你大致会达到：

- 能看懂并写出基础到中级代码
- 能做出结构清晰的小型控制台项目
- 能解释常见语法、类、函数、集合、异常、文件操作
- 能自己定位常见 bug
- 面对初中级面试题时不再完全发懵

## 九、怎么安排节奏最合适

建议不要贪快。

推荐节奏：

- 每次学 1 节
- 基础课可以一天 1 到 2 节
- 进阶和调试课建议一天 1 节
- 每周至少留 1 次复盘，把前面学过的例子自己重写

## 十、我给你的建议

你现在最该练的不是“刷很多内容”，而是：

- 每节真正看懂
- 每节都动手改
- 每节都练调试
- 每节都能讲出来

如果你能坚持这样学，这套课会比单纯看视频更扎实。
